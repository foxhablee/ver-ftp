# ТЗ на дизайн и разработку FTP клиента MVP версии
ver.FTP клиент - программное обеспечение, позволяющие создавать подключения к удаленным устройствам
с целью манипуляции над файловой структурой.
Включает в себя функционал с сохранением версий файлов и директорий при манипуляциях с ними.
Функциональность сохранения версий файлов реализуется с помощью интеграции локального git с
возможностью подключать к удаленным git серверам.

## Словарь

**ОКМ** — Основная Кнопка Мыши, по умолчанию Левая Кнопка Мыши.

**ВКМ** — Второстепенная Кнопка Мыши, по умолчанию Правая Кнопка Мыши.

**Удалённое устройство** — сервер или NAS, к которому осуществляется подключение.

**Локальное устройство** — устройство, с которого агент осуществляет подключение.

### Главное окно

Контейнер, объединяющий
[верхнее меню](#верхнее-меню),
[статусную строку](#статусная-строка),
[панель инструментов](#панель-инструментов)

[вкладки](#вкладка).

Главное окно может порождать [дочерние окна](#дочернее-окно).

### Дочернее окно

Окно, порождаемое [главным окном](#главное-окно) в процессе работы программы.
Не содержит [верхнего меню](#верхнее-меню), [статусной строки](#статусная-строка) и [панели инструментов](#панель-инструментов).
Количество дочерних окон, связанных с главным окном, не ограничено.

### Диалоговое окно

Диалоговое окно открывается поверх всех окон с вариантами действий в виде кнопок.
Пока не будет выбрано действие, нельзя воспользоваться программой.

* Инструменты

  * [Закладки](#закладки)

#### Уведомления

Иконка для открытия [списка уведомлений](#список-уведомлений).

#### Прогресс передачи файлов

Отображается при активных передачах.
Показывает скорость передачи и счётчик вида «2/10» (передано/всего).
Нажатие на элемент открывает [дочернее окно](#дочернее-окно) с активными передачами.

#### Статус текущего подключения

Статус подключения определяется последним кликом по [файловому менеджеру](#файловый-менеджер) с удалённым источником.
Клик по менеджеру с локальными файлами не меняет отображаемый статус.
Отображается иконкой:
- «подключено»
- «в процессе подключения» — соединение устанавливается или переподключение после разрыва
- серая иконка — активный менеджер отображает локальные файлы или заглушка

#### Выделенные элементы

Отображается при наличии выделения в активном [файловом менеджере](#файловый-менеджер).
Показывает общее количество выделенных элементов.
Если выделены и файлы, и директории — количество каждого типа отображается отдельно с иконками.

### Панель инструментов

Графически выделенные колонки на левой и правой стороне [главного окна](#главное-окно).
Содержат **кнопки инструментов** с иконками.
При наведении на _кнопку инструмента_ отображается всплывающая подсказка с названием инструмента.
Нажатие на кнопку раскрывает [окно инструментов](#окно-инструментов) внутрь главного окна со стороны панели:
левая панель раскрывает инструмент вправо от себя, правая — влево от себя.
Если на той же стороне уже открыт другой инструмент, он автоматически скрывается, уступая место новому.

По умолчанию в левой панели размещён инструмент [**Закладки**](#закладки).
В правой панели размещены инструменты [**Подключения**](#подключения) (открыт по умолчанию) и [**История Git**](#история-git).

### Окно инструментов

В раскрытом состоянии имеет сверху заголовок с названием и правее две кнопки:
скрытия и вызова контекстного меню.

Контекстное меню включает пункты нескольких типов по порядку:
1. Специфичные пункты, зависящие от конкретного инструмента.
2. Общие для всех инструментов:
- Переместить
  - Налево
  - Направо
- Убрать из панели инструментов

### Контент

Область [главного окна](#главное-окно) между панелями инструментов. Содержит [вкладки](#вкладка).

### Вкладка

Вкладка — рабочая поверхность, разделённая вертикально на две равные части.
Каждая часть содержит один [**файловый менеджер**](#файловый-менеджер).
Вкладки отображаются списком заголовков сверху [контента](#контент);
активная вкладка показывает своё тело, остальные — только заголовки.

Заголовок вкладки содержит название и кнопку закрытия.
Название формируется автоматически из имён двух файловых менеджеров в ней.

При запуске без подключений открывается одна вкладка с двумя файловыми менеджерами локальных файлов.

При закрытии вкладки все подключения, открытые в её файловых менеджерах,
закрываются во всех вкладках, где они используются.
В затронутых файловых менеджерах отображается [экран ошибки подключения](#ошибка-подключения).

### Всплывающее уведомление

Всплывающее уведомление — краткое сообщение о событии в программе.
Появляется в углу [главного окна](#главное-окно) и исчезает через несколько секунд.
Не блокирует работу с программой.

Существуют типы уведомлений:
- **Информационное** — сообщает о событии, не требует действий.
- **Ошибка** — то же самое как информационное, выделяется красным цветом на заднем фоне.

Типы событий, порождающих уведомление:
- Подключение установлено _(Информационное)_
- Подключение разорвано _(Информационное)_
- Ошибка подключения _(Ошибка)_
- Передача файлов завершена _(Информационное)_
- Ошибка передачи файлов _(Ошибка)_

#### Список уведомлений

Кнопка в [статусной строке](#статусная-строка) открывает [дочернее окно](#дочернее-окно)
со списком всех уведомлений за текущую сессию в обратном хронологическом порядке.
Каждый элемент списка отображает:
- Заголовок уведомления
- Краткое описание
- Время

## Файловый менеджер

Файловый менеджер — панель со списком файлов и директорий текущей директории.

### Заголовок-переключатель

Кнопка в верхней части файлового менеджера с текущим названием источника.
По нажатию раскрывается контекстное меню со списком вариантов:
- Локальные файлы
- Активное подключение текущей [вкладки](#вкладка) _(если есть)_
- Подключения из других вкладок _(файловый менеджер переключается на ту же сессию)_
- Новое подключение

При выборе подключения из другой вкладки файловый менеджер переключается на ту же сессию —
несколько менеджеров используют один экземпляр подключения, но могут отображать разные директории.

При выборе «Новое подключение» предварительно открывается [диалоговое окно](#диалоговое-окно) создания подключения.

### Кнопки с действиями

После заголовка первыми расположены кнопки с быстрыми действиями:
- **Обновить** — перезагружает содержимое текущей директории.
- **Создать файл**
- **Создать папку**

### Хлебные крошки

Под кнопками хлебные крошки с текущим путём — по нажатию переходит в редактируемое поле.

### Список элементов

Отображает файлы и директории текущей директории.
Первым элементом списка отображается `..`.
При двойном нажатии переходит в предыдущую директорию.
В корневой директории элемент `..` не отображается.

Переход в директорию осуществляется двойным кликом ОКМ по ней.

Список выполнен в виде таблицы с колонками:
- Название
- Размер _(у директорий не отображается)_
- Тип
- Дата изменения

По умолчанию список сортируется по типу и названию:
сначала директории в алфавитном порядке, затем файлы в алфавитном порядке.
Ручная сортировка по колонкам недоступна.

Элементы можно выделять:
- Зажатием ОКМ и перемещением курсора (выделение прямоугольником)
- Ctrl+клик для добавления элемента к выделению

### Контекстное меню

При ВКМ на элементе открывается контекстное меню:
- Переименовать
- Скопировать
- Вырезать
- Удалить
- Скачать _(только на удалённом устройстве)_
- Загрузить _(только на локальном устройстве)_

При ВКМ на пустом месте:
- Создать файл
- Создать папку
- Вставить _(недоступно если буфер обмена пуст)_

При выделении нескольких элементов пункты применяются ко всем выделенным.

#### Переименовать

При выборе «Переименовать» название элемента переходит в режим инлайн-редактирования.
Подтверждение — Enter, отмена — Esc.

#### Удалить

При выборе «Удалить» открывается [диалоговое окно](#диалоговое-окно) с подтверждением удаления.

### Создание файла и папки

Над хлебными крошками расположены кнопки **«Создать файл»** и **«Создать папку»**.
Те же пункты доступны в контекстном меню при ВКМ на пустом месте списка.

При нажатии в списке элементов появляется новый элемент с активным полем ввода названия.
При отмене ввода элемент исчезает.
При вводе названия и подтверждении через Enter элемент создаётся.

### Ошибка подключения

Если подключение разорвано, программа автоматически пытается переподключиться с теми же параметрами.
На время попытки тело файлового менеджера заменяется экраном с индикатором загрузки.
Если переподключение невозможно — отображается экран ошибки с кнопками:
- **Переподключиться** — повторная попытка с теми же параметрами
- **Сменить источник** — открывает [заголовок-переключатель](#заголовок-переключатель)

## Передача файлов

Передача файлов между файловыми менеджерами осуществляется через копировать/вырезать и вставить.
Drag-and-drop между файловыми менеджерами не поддерживается.
Передача не блокирует работу с программой.
Несколько передач выполняются параллельно.

### Прогресс передачи

При нажатии на индикатор прогресса в [статусной строке](#статусная-строка)
открывается [дочернее окно](#дочернее-окно) со списком активных передач.

Каждый элемент списка отображает:
- Название файла
- Направление передачи
- Прогресс в процентах
- Скорость передачи
- Кнопка отмены передачи

### Ошибка передачи

При ошибке передачи появляется [всплывающее уведомление](#всплывающее-уведомление)
типа «Ошибка» с названием файла и причиной ошибки.

### Подключения

Окно инструмента для управления сохранёнными подключениями.

Специфичные пункты контекстного меню:
- Закрыть все подключения

#### Создание подключения

Первый элемент окна — кнопка **«Создать новое подключение»**.
По нажатию открывается [дочернее окно](#дочернее-окно) с полями:

- Название
- Адрес
- Порт
- Протокол
- Имя пользователя
- Пароль
- Чекбокс сохранения пароля _(по умолчанию не активирован)_
- Чекбокс сохранения подключения _(по умолчанию активирован)_

Доступные протоколы: FTP, SFTP, FTPS.

Поле адреса принимает произвольную строку — IP-адрес или доменное имя.
Порт — число от 0 до 65535.
Если порт введён, но протокол не выбран, протокол определяется автоматически:
- 21 — FTP
- 22 — SFTP

Поле пароля скрывает символы с возможностью их отобразить.

#### Список подключений

Под кнопкой создания расположен список сохранённых подключений.
Каждый элемент отображает название подключения.

При нажатии на элемент подключение открывается в активном [файловом менеджере](#файловый-менеджер).
При наведении на элемент подключения появляются кнопки:
- **Редактировать** — открывает [дочернее окно](#дочернее-окно), аналогичное окну [создания подключения](#создание-подключения), с заполненными полями
- **Удалить** — открывает [диалоговое окно](#диалоговое-окно) с подтверждением удаления

### История Git

Окно инструмента со списком коммитов активного [файлового менеджера](#файловый-менеджер) с удалённым источником.
Активный источник определяется последним кликом по файловому менеджеру.
Клик по файловому менеджеру с локальными файлами не меняет отображаемый источник истории.

Если ни один файловый менеджер не отображает удалённый источник —
показывается история от последнего активного менеджера с удалённым источником за текущую сессию.
Если такого не было — отображается пустой список с заглушкой.

Специфичные пункты контекстного меню:
- Обновить

#### Принцип работы

При создании подключения программа создаёт локальную директорию в своём git-репозитории.
Git-репозиторий хранится только локально в программе, на удалённом сервере `.git` папки не создаются.

Любая операция с файлами и директориями удалённого источника через программу:
1. Загружает содержимое изменённых файлов в локальную директорию
2. Создаёт коммит
3. Синхронизирует локальный git с удалённым git-репозиторием через force push.
   Force push используется для поддержания единой линейной истории —
   при конфликте версий побеждает последняя запись.

Операции с целой директорией (перемещение, замена, удаление) фиксируются одним коммитом.

При получении метаданных файлов для отображения в [файловом менеджере](#файловый-менеджер) содержимое файлов не загружается.
Содержимое загружается только при операциях с файлами (замена, удаление, перемещение).

#### Отслеживание внешних изменений

Программа не размещает git на удалённом сервере, поэтому изменения, внесённые напрямую на сервер
без программы, не фиксируются автоматически.

При открытии директории, создании подключения, обновлении директории и любых операциях,
обновляющих отображаемые данные удалённого сервера — программа проверяет изменения через локальный git.
Если обнаружено удаление или появление нового файла — создаётся коммит от безымянного автора.
Изменение содержимого существующего файла через внешние источники в MVP не отслеживается.

Коммиты от безымянного автора отображаются в [истории](#список-коммитов) без возможности отката.

#### Структура окна

Сверху отображается название подключения из сохранённого доступа.

Ниже — строка поиска по тексту коммита и кнопка фильтрации по дате правее.
При нажатии на кнопку фильтрации открывается [дочернее окно](#дочернее-окно) с выбором даты и кнопками подтверждения и отмены.
Если фильтр по дате активен — рядом с кнопкой фильтрации появляется кнопка очистки фильтра.

#### Список коммитов

Каждый элемент отображает:
- Хэш коммита
- Автор
- Дата
- Сообщение коммита

Контекстное меню элемента:
- Скопировать хэш
- Вернуть _(недоступно для коммитов от безымянного автора)_

Список загружается порциями по мере прокрутки (ленивая загрузка).

#### Откат файлов

При выборе пункта «Вернуть» открывается [дочернее окно](#дочернее-окно) со списком файлов, затронутых в выбранном коммите.

Список выполнен в виде таблицы с колонками:
- Выбор (чекбокс)
- Название
- Директория
- Тип операции _(создан / изменён / удалён)_

Над списком кнопки:
- Отметить всё
- Деактивировать всё

Под списком кнопки:
- Применить
- Отмена

После подтверждения выбранные файлы откатываются до состояния из выбранного коммита,
изменения применяются на сервере и создаётся новый коммит.
Файлы, удалённые в откатываемом коммите, восстанавливаются на сервере.
Файлы, созданные в откатываемом коммите, удаляются с сервера.

### Закладки

Окно инструмента со списком сохранённых директорий.
Позволяет быстро открывать нужные директории.

#### Быстрые действия

Над списком расположена кнопка **«Добавить закладку»**.
При нажатии открывается [дочернее окно](#дочернее-окно) с полями:
- Название
- Путь

Нельзя создать две закладки с одинаковым путём.

#### Список закладок

Плоский список без вложенности, отсортированный по алфавиту.
Каждый элемент отображает:
- Название
- Абсолютный путь
- Иконка источника (локальный или удалённый)

По двойному нажатию ОКМ на закладку открывает директорию в активном [файловом менеджере](#файловый-менеджер).
Если закладка относится к удалённому устройству и подключение не установлено — создаётся новое подключение в новой [вкладке](#вкладка).

#### Контекстное меню закладки

- **Удалить** — открывает [диалоговое окно](#диалоговое-окно) с подтверждением удаления.
- **Редактировать** — открывает [дочернее окно](#дочернее-окно), аналогичное окну создания закладки, с заполненными полями.
- **Переименовать** — название переходит в режим инлайн-редактирования.
  Подтверждение — Enter, отмена — Esc.

## Настройки

Открываются через Файл → Настройки. [Дочернее окно](#дочернее-окно) с вкладками.

### Уведомления

- Поле ввода: время показа [всплывающего уведомления](#всплывающее-уведомление) в секундах

### Оформление

- Переключатель темы: светлая / тёмная

### Git

Настройка удалённого репозитория для синхронизации локального git программы.

Поля:
- URL удалённого репозитория
- Имя пользователя
- Токен
- Пароль

Если remote не настроен — локальный git работает без синхронизации, force push не выполняется.

## Сочетания клавиш

| Сочетание | Действие |
|-----------|----------|
| Ctrl+T | Открыть новую [вкладку](#вкладка) |
| Ctrl+W | Закрыть текущую вкладку |
| Ctrl+C | Скопировать выделенные элементы в буфер обмена |
| Ctrl+V | Вставить элементы из буфера обмена в активный [файловый менеджер](#файловый-менеджер) |
| Ctrl+X | Вырезать выделенные элементы |
